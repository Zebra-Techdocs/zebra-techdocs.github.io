Using Zebra DNA Cloud

PRE-RELEASE DOCUMENTATION

Overview

After one or more devices are enrolled and licensed in the zDNA Cloud system, administrators can begin creating and deploying device settings and performing other administrative tasks. These tasks are performed by administrators as defined below:

  • Tenant Owner - assigned to the first account created along with a new Zebra DNA Cloud instance; requires registration at Zebra.com; uses Zebra SSO credentials; able to create zDNA Administrative Users.
  • Administrative Users - can access all features of zDNA Cloud except user administration; created and maintained by the Tenant Owner.

Requirements

  • One or more Zebra devices running Android 11 (or later) enrolled, licensed
  • Obtained during zDNA Cloud Setup:
    • Zebra.com login credentials
    • A zDNA Cloud Tenant Owner account

Manage Devices

Devices can be managed in the zDNA Cloud system only after they have been enrolled and licensed in the system. To enroll devices in zDNA Cloud for the first time, see Enroll Devices in the Setup Section.

Device Actions

On Individual Devices

From the My Devices screen, the following actions can be executed remotely on enrolled device(s): Click image to enlarge; ESC to exit.

  • Reset Device Passcode - removes any existing swipe pattern, PIN or password
  • Reboot Device - causes all apps to quit and restarts the device
  • Wipe Device - resets the device to its factory-fresh state, permanently erasing all user apps and data
  • Delete Device - removes zDNA client app from device, de-enrolls device from zDNA Cloud system; other apps and settings on device remain unchanged ✝︎
  • Update zDNA Client - checks the client app on the device and pushes an update, if necessary

✝︎ Deleted devices could remain visible in deployment detail documents if the device was selected for deployment before being deleted.

Device Groups

To perform actions on a group of devices all at once, it can be helpful to create one or more device Groups and associate certain devices with that Group. For example, a Group called "Warehouse" might be created to configure devices to connect with the access points in the company's warehouse and to receive its apps for inventory and order picking. Devices added to a Group automatically receive the configuration settings created for that Group. Groups can be created and devices added during enrollment or at any time thereafter.

Click on the pencil icon in a device's row to manage the Group it belongs to.
This action might effect device licensing. A device can belong to NO MORE than one Group at a time. Click image to enlarge; ESC to exit.

Manage Settings and Groups

As in Zebra StageNow, Zebra DNA Cloud employs the concept of Profiles, which are collections of device settings and/or apps to configure devices to behave in a certain way or to enable workers to perform particular sets of tasks. Zebra DNA Cloud also implements device Groups, allowing administrators to deploy a Profile associated with that Group simply by adding devices to the Group.

Create a Settings Profile

To begin, go to the Device Settings section and select the settings type under the "New Settings" button:
    • Wizard - Presents configuration parameters with the most common options pre-selected (shown with Step 1 below)
    • Advanced - Presents all available configuration parameters with default options selected (shown with Advanced Settings section below)
Click image to enlarge; ESC to exit.

Use Settings Wizard

This section presents the most common configuration parameters with common options pre-selected.

  1. Configure Language/Locale and Date/Time settings as desired and click "Next" button: Click image to enlarge; ESC to exit.

  2. Configure Networks as desired and click "Next" button: Click image to enlarge; ESC to exit.

  3. Configure Security settings as desired and click "Next" button: Click image to enlarge; ESC to exit.

  4. Configure Display settings as desired and click "Next" button: Click image to enlarge; ESC to exit.

  5. Configure other hardware settings as desired and click "Next" button: Click image to enlarge; ESC to exit.

  6. Enter a Name for this settings Profile or accept the auto-generated one.
    Add a short description to help identify the Profile later.
    Click "Next" button when finished: Click image to enlarge; ESC to exit.

  7. Review all settings and edit as needed.
    Click "Apply Now" button to proceed: Click image to enlarge; ESC to exit.

  8. Select target device(s) and/or Groups to receive the profile.
    Click "Apply" to deploy immediately: Click image to enlarge; ESC to exit.

  9. Monitor progress of settings deployments in the Device Settings landing page: Click image to enlarge; ESC to exit.

Selecting a Group in Step 9 above applies all settings created in the Profile to:
   • All devices currently in that Group
   • Devices added to the Group at any time in the future

Manage Apps

This section covers installation and uninstallation of apps on devices and deployment of related files, if any.

Install app(s) and/or files

1. In the My Apps section, click on the "Configure and Deploy App(s)" button: Click image to enlarge; ESC to exit.

2. Enter the package name and source URL for the app being installed.
To deploy only files (and no app), uncheck the "App Installation" box and proceed to Step 3. Click image to enlarge; ESC to exit.

3. If adding files, click the "Add app-related files" button to add files for deployment.
Click "Add" when finished adding each file name and desired destination on the device.
Repeat until all related files are added: Click image to enlarge; ESC to exit.

4. Click "Next" to proceed: Click image to enlarge; ESC to exit.

5. Select the target device(s) to receive the app and/or file(s).
When finished selecting target(s), click "Apply Now" to push changes: Click image to enlarge; ESC to exit.

6. Confirm app deployment to device(s) by individual inspection or user confirmation.

Manage Users

The Tenant Owner creates Administrative Users in the zDNA Cloud system using Zebra.com credemntials. Admin users can perform all tasks within the system except user administration.

This section requires login as the Tenant Owner using Zebra.com credentials. For credentials, please register with Zebra.com.

Create Admin Users

1. Point browser to Zebra DNA Cloud server and log in as "Tenant Owner" using Zebra.com credentials: Click image to enlarge; ESC to exit.

2. Click user icon for menu and select "Manage All Users" from the menu: Click image to enlarge; ESC to exit.

3. In All Users screen, click "New User" button: Click image to enlarge; ESC to exit.

4. Enter user info and click "Add" button: Click image to enlarge; ESC to exit.

5. New user appears in All Users list:
Click image to enlarge; ESC to exit.

6. Instruct the new user to visit zDNA Cloud server and login as an Administrative User.

7. At login, select "Forgot Password" to generate an email with further instructions.

8. Repeat this process for all Administrative Users.
