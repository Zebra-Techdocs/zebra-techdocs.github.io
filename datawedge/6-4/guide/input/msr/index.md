<h2 id="overview">Overview</h2>
<p><strong>Magnetic Stripe Reader (MSR) Input</strong> is used to acquire data from a "mag-stripe" card such as a credit or debit card. The information stored on the black magnetic strip is transferred to the device when the card is "swiped" through the MSR. Information is encoded in an open format using the ANSI x4.16 standard and usually relates to the carrier's identity and/or financial institution. DataWedge can acquire and output the raw data from most cards that use the ANSI x4.16 format. For cards that also adhere to <a href="https://www.magtek.com/content/documentationfiles/d99800004.pdf">ISO/ABA specifications</a> published by the ISO and American Bankers Association, developers can opt to enable Zebra encryption, which protects data immediately upon acquisition and at all times thereafter. Once enabled on a Zebra device, <strong>encryption cannot be disabled</strong>. </p>
<p>Under the <a href="https://www.magtek.com/content/documentationfiles/d99800004.pdf">ANSI magnetic stripe standard</a>, data is stored on the card in three tracks. Unencrypted data can be acquired from any of the individual tracks or from all three at once. The same is true of data encrypted using Zebra's Enhanced Mode encryption. Zebra Original Mode encryption treats all three tracks as a single entity.  </p>

<em>The three modes of MSR card encoding, two of which include encryption</em>
<br>

<h4 id="enablingencryption">Enabling Encryption</h4>
<p><strong><u>Enabling encryption on a device cannot be undone</u></strong>. Once a device has been configured to encrypt ISO/ABA data, it can never again be configured to output raw ISO/ABA data; switching between Original and Enhanced encryption modes is the only permitted change on such devices. <u>Encryption applies only to ISO/ABA data</u>; all other MSR-acquired data is output in unencrypted form using Non-ISO Mode. </p>
<p>To learn how to enable encryption and work with encrypted data, refer to the <a href="https://atlassian.idtechproducts.com/confluence/download/attachments/30479625/80101505-001-H%20User%20manual%20SecureHead%20USB%20UART.pdf?api=v2">ID TECH SecureHead User Manual</a>. </p>

<h2 id="msroutput">MSR Output</h2>
<p><strong><u>DataWedge outputs MSR data only through intents</u></strong>. This is true whether acquired data is open or encrypted. It is therefore necessary for <!--Intent Output to be enabled and-->the receiving application to be capable of accepting and processing data from an intent bundle. To help simplify the use of MSR data for application developers, DataWedge automatically parses acquired data, formats it for specific uses, and places the modified data into specific tags. For example, the code for printing a receipt might make use of one of the "Masked Data (ASCII)" tags, which displays all but the last four digits of the credit card number as asterisks. </p>
<p>For modifying the acquired data, DataWedge offers only the formatting options shown in the sections below. Options for formatting data beyond those shown can be found in the <a href="https://atlassian.idtechproducts.com/confluence/download/attachments/30479625/80101505-001-H%20User%20manual%20SecureHead%20USB%20UART.pdf?api=v2">ID TECH SecureHead User Manual</a>. <strong>Custom data formatting is the sole responsibility of the developer</strong>.</p>

<h2 id="nonisomode">Non-ISO Mode</h2>
<p>Non-ISO Mode is the default mode on any device on which encryption is not enabled. This mode outputs the data without encryption exactly as read form the card. Its three status tags (in the Mapping table below) indicate the presence of data in each of the corresponding tracks on the card. </p>
<p>On devices with encryption enabled, Non-ISO Mode is invoked only when reading a card that is <strong><u>not</u></strong> ISO/ABA compliant; data from such cards is otherwise encrypted using Original or Enhanced Mode and can never be configured otherwise. </p>
<p><strong>MSR data from <u>all three tracks</u> is contained in the tag</strong>: <code>com.symbol.datawedge.msr_data</code></p>

<h2 id="originalmode">Original Mode</h2>
<p>Zebra Original Mode encrypts data on the three tracks as a single entity, preventing track data from be handled separately except as contained in the tags described below, which Zebra created as a convenience according to the most common use cases. All readable data from all three tracks on the card is placed in the tag <code>com.symbol.datawedge.msr_data</code> and can be parsed as desired by the developer.  </p>
