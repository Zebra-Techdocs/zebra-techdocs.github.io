Used to select an Action to be performed on a file or folder (if empty) on the device.

IMPORTANT

In preparation for Google-mandated security restrictions being phased in starting with devices running Android 11, Zebra has implemented Secure Storage Manager (SSM), a mechanism for securely sharing files and/or data from an originating app with one or more target apps in compliance with Android security mandates. SSM is preinstalled on all Zebra devices running Android 11 and later.

SSM is compatible with the File Manger CSP delivered with MX 11.3 (and later) and was designed to be completely transparent to the administrator in many file- and/or data-sharing scenarios. In some cases, minor updates are required to apps designated as the recipient of files and/or data shared through SSM. Such changes would be required not specifically for working with Zebra devices, but for compliance with Android security mandates (and will eventually be required of all Android devices).

Required Parameters

FileAction Option 10 requires (or optionally works with, as indicated) these additional parameters:

  • TargetApplicationSignature (required) proves "target" app (for which file is provided) is genuine
  • SourceAccessMethod2 (required) provides location of file to be copied/transferred to device
    For added security, Zebra recommends selecting the "File on remote server" option (see SourceURI2, below)
  • SourceURI2 provides server location of file being copied (required for "File on remote server")
    SourceURI2 option requires apps to be capable of consuming data via URI
  • PersistFile (optional) allows copied data and/or file to securely persist following an Enterprise Reset
